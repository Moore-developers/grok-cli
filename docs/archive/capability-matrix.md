# Grok 能力矩阵

## 目标

列出 Hermes 当前通过共享 xAI 凭据接上的 Grok 能力面，方便后续 SKILL
能够把它们统一纳入一个可使用的能力路由层。

## 能力矩阵

| 能力 | Hermes 中的能力面 | 是否复用共享 OAuth | 说明 |
| --- | --- | --- | --- |
| 聊天 / 推理 | xAI OAuth 模型 provider | 是 | Grok 主运行路径 |
| 工具调用 | Grok 聊天运行时 | 是 | 取决于模型和运行时支持 |
| 流式输出 | Grok 聊天运行时 | 是 | 属于主运行时传输能力 |
| X 搜索 | `x_search` | 是 | 使用 xAI 内建 Responses 工具 |
| 图片生成 | xAI 图像 provider | 是 | Grok 图片模型 |
| 视频生成 | xAI 视频 provider | 是 | 文生视频和图生视频 |
| 文本转语音 | xAI TTS | 是 | `/v1/tts` |
| 语音转文字 | xAI STT | 是 | `/v1/stt` |
| OpenAI 兼容代理 | xAI proxy adapter | 是 | 次级消费能力面 |

## 能力说明

### 聊天 / 推理

这是最核心的 Grok 模式。当用户只说“用 Grok”而没有明确指定媒体或搜索类
任务时，默认应当走这条路径。

### X 搜索

这不是普通网页搜索，而是一个专门面向 X/Twitter 的搜索能力面，背后依赖
xAI 服务端内建工具支持。

### 图片生成

支持 Grok 专属图像模型，应当被视为一等能力，并纳入统一路由。

### 视频生成

支持纯提示词生成，也支持基于输入图片进行动画化生成，应纳入统一路由。

### 文本转语音

未来的 SKILL 应当能够把“用 Grok 读出来”这类请求，在 OAuth 检查通过后，
路由到 Grok TTS 能力面。

### 语音转文字

未来的 SKILL 应当能够把“用 Grok 转写这段音频”这类请求，在 OAuth 检查
通过后，路由到 Grok STT 能力面。

### 代理

这不是一个直接面向最终用户的媒体技能，而是一种桥接能力，用来让其他工具
通过 OpenAI-compatible 形状来消费 Grok。
